Layered Web API for jsonschema.zig

Three layers: start as close to std.http.Server as possible, then opt into route registration for OpenAPI, then opt into full request and response contracts for the most declarative API.

Rule: OpenAPI generation requires route registration. Helper-only usage can validate requests and responses, but it does not provide enough route metadata to produce an OpenAPI document.

Supported Layers

Layer 1

Helper Functions

Mix validation into existing stdlib handlers.

Use for: easiest migration from raw stdlib.

  • Caller keeps the accept loop, routing, and raw request.
  • Helpers read, parse, validate, and respond with JSON.
  • No OpenAPI without additional route registration.
Code 
if (req.head.method == .POST and web.pathEql(&req, "/users")) {
    const input = try web.jsonBody(CreateUser, arena, &req, .{});
    const user = try createUser(&store, input);
    return web.respondJson(User, arena, &req, .created, user, .{});
}
Layer 2

Route Table + Markers

Register routes and infer schemas from handler signatures.

Use for: FastAPI-like ergonomics and OpenAPI.

  • Method and path are explicit.
  • Body, status, and response schema come from marker types.
  • Route table is the source for OpenAPI generation.
Code 
const Api = web.Api(.{
    web.post("/users", createUser),
});

fn createUser(ctx: *Ctx, body: web.Body(CreateUser)) !web.Created(User) {
    return .{ .value = try ctx.store.create(body.value) };
}
Layer 3

Request/Response Contracts

Explicit operation types for body, query, path, success, and errors.

Use for: complete OpenAPI accuracy.

  • Best for multiple responses and error bodies.
  • More verbose, but fully declarative.
  • Can still dispatch to ordinary handler functions.
Code 
const CreateUserRequest = web.Request(.{
    .body = CreateUser,
});

const CreateUserResponse = web.Response(.{
    web.case(.created, User),
    web.case(.bad_request, Problem),
    web.case(.unprocessable_entity, ValidationProblem),
});

const Api = web.Api(.{
    web.post("/users", createUser, .{
        .request = CreateUserRequest,
        .response = CreateUserResponse,
    }),
});

Comptime OpenAPI

What can be built at comptime

  • Route method/path/status metadata from Layer 2 or 3.
  • Request and response schemas from Zig types.
  • OpenAPI operation shapes from handler or contract types.
  • Compile errors for unsupported route or schema shapes.

What stays runtime

  • Network I/O and std.http.Server.Request handling.
  • Reading request bodies with size limits.
  • Parsing and validating actual request/response values.
  • Serving the generated OpenAPI JSON or Swagger UI route.

Base Model Equivalent

Zig has no superclasses

Models stay plain structs. Shared behavior is expressed with declarations, composition, and generic type factories instead of inheritance.

Model metadata 
const CreateUser = struct {
    name: []const u8,

    pub const jsonschema = .{
        .fields = .{
            .name = .{ .minLength = 1 },
        },
    };
};

Use contract wrappers

Request and response wrappers act like FastAPI/Pydantic base models: they attach HTTP meaning to ordinary Zig structs at comptime.

Response cases 
const CreateUserResponse = web.Response(.{
    web.case(.created, User),
    web.case(.bad_request, Problem),
    web.case(.unprocessable_entity, ValidationProblem),
});

HTTP Pass-Through

Contract enhances JSON

JSON is where the library adds schema emission, validation, typed parsing, typed serialization, and OpenAPI schema generation.

JSON response 
fn createUser(ctx: *Ctx, body: web.Body(CreateUser)) !web.Created(User) {
    return .{ .value = try ctx.store.create(body.value) };
}

Everything else passes through

HTML, text, bytes, blobs, files, redirects, streaming, WebSockets, and custom headers should use the underlying std.http.Server.Request APIs directly or through tiny convenience helpers.

Pass-through response 
fn download(req: *std.http.Server.Request) !void {
    try req.respond(bytes, .{
        .status = .ok,
        .extra_headers = &.{
            .{ .name = "content-type", .value = "application/octet-stream" },
        },
    });
}

Minimal Surface

// Layer 1
pub fn jsonBody(comptime T: type, arena: Allocator, req: *Request, opts: BodyOptions) !T;
pub fn respondJson(comptime T: type, arena: Allocator, req: *Request, status: Status, value: T, opts: ResponseOptions) !void;
pub fn body(req: *Request, arena: Allocator, opts: BodyOptions) ![]const u8;

// Layer 2
pub fn Body(comptime T: type) type;
pub fn Created(comptime T: type) type;
pub fn Status(comptime status: std.http.Status, comptime T: type) type;
pub fn get(comptime path: []const u8, comptime handler: anytype) Route;
pub fn post(comptime path: []const u8, comptime handler: anytype) Route;
pub fn Api(comptime routes: anytype) type;

// Layer 3
pub fn Request(comptime spec: anytype) type;
pub fn case(comptime status: std.http.Status, comptime T: type) ResponseCase;
pub fn Response(comptime cases: anytype) type;

// OpenAPI
pub fn openApiJson(comptime Api: type, allocator: Allocator, opts: OpenApiOptions) ![]u8;
pub fn writeOpenApi(comptime Api: type, writer: *std.Io.Writer, opts: OpenApiOptions) !void;

Bottom line: Layer 1 preserves the stdlib. Layer 2 unlocks OpenAPI with low ceremony. Layer 3 covers serious APIs with explicit error and response contracts.
